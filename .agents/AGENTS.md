# Custom Agent Rules

## Website Content Management System (CMS)
For all web application or website projects, always implement a visual Content Management System (CMS) or admin panel (e.g. at `/cms` or `/admin`) secured with a login gate. This system must allow non-technical administrators to visually edit, add, and delete website content, settings, parameters, and company metadata (such as company details, contact info, rules, and limits) without having to modify HTML, database, or backend files directly.

## 2026 Frontend Design System & Style Guidelines
For any website, client landing page, or product dashboard built, enforce the following style and architecture patterns:
1. **Visual Styles**:
   - **Minimal but expressive UI**: Clean layouts, strong typography, selective highlight color, no distractions.
   - **Retro / Brutalist**: Asymmetric layouts, visible grids, thick solid borders, flat primary color swatches.
   - **Soft Depth & Layered Surfaces**: Soft translucency/glassmorphism, subtle shadows, backdrop blurs, organic layered surfaces.
   - **Bold Gradients**: Saturated gradient color shifts, dual-tones.
   - **Nostalgic**: Tactile textures, 90s vintage bevel details.
2. **Interaction Styles**:
   - **Micro-interactions**: Tiny, snappy hover, active, loading, and successful state animations.
   - **Motion-first**: User journeys guided by structured transitions.
3. **Principles**:
   - Keep UI responsive, lightweight, clean, and fast.
   - Limit heavy animations only to parts where it clarifies information layout.
   - Build accessible interfaces by default with strong color contrast, clear typographic hierarchy, and semantic structures.
